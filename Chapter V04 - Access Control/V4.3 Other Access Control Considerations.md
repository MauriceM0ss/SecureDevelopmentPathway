# V4.3 Other Access Control Considerations

## 4.3.1 Verify administrative interfaces use appropriate multi-factor authentication to prevent unauthorized use

Omdat wachtwoorden op zichzelf niet meer als veilig worden gezien is het belangrijk dat belangrijke omgevingen zoals bijvoorbeeld een administrator portaal MFA heeft. Denk hierbij aan bijvoorbeeld een code die je na het invullen van het wachtwoord nog moet opgeven.

### Hoe te testen?

#### Optie 1 (manueel)

Controleer dat er een MFA check zit bij het inloggen op een administrator portaal.

### Hoe te implementeren?

* In de meest gevallen is dit via een configuratie bestand in te stellen.
* Indien er via een authenticatie API wordt gewerkt gebruik de functionaliteit beschikbaar voor MFA.
* Ga nooit een eigen implementatie schrijven!

Voor meer informatie zie [Best Practice - MFA](../Best%20Practices/Best%20Practice%20-%20MFA.md)

## 4.3.2 Verify that directory browsing is disabled unless deliberately desired. Additionally, applications should not allow discovery or disclosure of file or directory metadata, such as Thumbs.db, .DS_Store, .git or .svn folders

Als directory browsing aan staat is het mogelijk om door de webapplicatie heen te klikken als een map structuur. Dit vergroot de kans dat gevoelige bestanden of source code uitlekken. Hetzelfde geldt voor metadata, op het moment dat de .git folder publiek beschikbaar is, is het in sommige gevallen mogelijk om de gehele history te zien waardoor secrets en source code mogelijk uitlekken.

### Hoe te testen?

#### Optie 1 (via tool)

Gebruik een Web Fuzzer tool zoals [wfuzz](https://github.com/xmendez/wfuzz) in combinatie met de lijsten beschikbaar op bijvoorbeeld [FuzzDB](https://github.com/fuzzdb-project/fuzzdb/tree/master) om te scannen voor bestanden die niet publiekelijk beschikbaar zouden moeten zijn.

#### Optie 2 (via Pentest)

Vraag een Pentest aan.

### Hoe te implementeren?

In de meeste gevallen is dit te configureren via de server:

* Apache <https://cwiki.apache.org/confluence/display/HTTPD/DirectoryListings>
