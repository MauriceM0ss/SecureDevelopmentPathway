# V13.3 SOAP Web Service

## 13.3.1 Verify that XSD schema validation takes place to ensure a properly formed XML document, followed by validation of each input field before any processing of that data takes place

Een XML Schema Definition (XSD) is nodig om de toegestane onderdelen van een XML document te definiÃ«ren:

* De elementen en attributen die mogen voorkomen in een document
* De hoeveelheden (en volgorde van) child elementen
* Data type voor elementen en attributen
* Default en vaste waarden voor elementen en attributen

Met implementeren van een XSD-schema-validatie, gevolgd door validatie van elk invoerveld kan je er voor zorgen dat er geen gemanipuleerde input verwerkt kan worden door je applicatie, zoals het ophalen van bestanden op je Apache server.

XSD is vergelijkbaar met een database schema, wat de data beschrijft in je database.

### Hoe te testen?

#### Optie 1 (via de codebase)

* Zoek naar codefragmenten die XML-gegevens verwerken.
* Controleer of er een verwijzing is naar een XSD-schema en of er validatie plaatsvindt tegen dit schema.

#### Optie 2 (manueel)

* Probeer handmatig een goed gevormd XML-bestand en een slecht gevormd XML-bestand in te dienen via de webpagina.
* Check of het systeem het slecht gevormde XML-bestand afwijst en het goed gevormde bestand accepteert.

#### Optie 3 (via tools)

Er zijn test-tools beschikbaar om XML-validatie te kunnen controleren, zoals [Online XML Validator (XSD)](https://www.liquid-technologies.com/online-xsd-validator) of [XML Validation](https://www.xmlvalidation.com/)
